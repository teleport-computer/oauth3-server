Asks with a scoped token, receives just the requested data — your timeline, your history — not the credential.

step by step

  1. Teleport your session in. You sign into the site (X, Reddit, YouTube…) in the enclave’s browser — the extension can hand off an existing session. The cookie is now sealed in the pod.
  2. It stays put. The browser reaches the site through an egress-locked proxy bound to that one domain. The credential can’t be exfiltrated and can’t be pointed elsewhere.
  3. An app asks for a read. It presents a scoped token (granted through your oauth3 dashboard) — “read my X timeline, nothing else.” It never sees the cookie.
  4. The container does the read. It navigates, renders, and returns only what was asked — a screenshot, the DOM, or a parsed result.
    SPI: /session · /navigate · /screenshot · /eval
  5. Anyone can check the code. The container is attested — its exact published build is measured, so a user (or the app) can verify it does only this before trusting it.

The property: the app gets the read, the enclave keeps the credential. One login, inside the sealed room, powers every scoped read after it — and the code doing it is verifiable, not trusted.
